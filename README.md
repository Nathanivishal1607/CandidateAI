## 🚀 Setup Instructions

### 1. Configure Environment

Edit `.env` with your actual values:

```env
ANTHROPIC_API_KEY=sk-ant-...        # Get from console.anthropic.com
DB_HOST=localhost
DB_PORT=3306
DB_NAME=candidate_search
DB_USER=root
DB_PASSWORD=your_mysql_password
PYTHON_SERVICE_URL=http://localhost:5001
TOMCAT_HOME=/opt/tomcat             # Path to your Tomcat installation
```

### 2. Set Up the Database

```bash
mysql -u root -p < db/schema.sql
mysql -u root -p candidate_search < db/seed.sql
```

### 3. Start the Python Service

```bash
cd python-service
pip install -r requirements.txt
python app.py
# Service starts on http://localhost:5001
```

> **Note:** First startup downloads the `all-MiniLM-L6-v2` model and PaddleOCR weights (~400 MB total). Subsequent starts are fast.

### 4. Build and Deploy the Java Backend

```bash
cd java-backend
mvn clean package

# Copy WAR to Tomcat
cp target/candidate-search.war $TOMCAT_HOME/webapps/

# Start Tomcat
$TOMCAT_HOME/bin/startup.sh
```

### 5. Open the App

Navigate to: **http://localhost:8080/candidate-search/**

## 🧪 API Reference

### Python Microservice

#### `GET /health`
```json
{ "status": "ok", "model": "all-MiniLM-L6-v2", "ocr": "PaddleOCR" }
```

#### `POST /score`
```bash
curl -X POST http://localhost:5001/score \
  -H "Content-Type: application/json" \
  -d '{"query":"Java developer SQL","resume_text":"Alice Java Spring SQL 3 years"}'
```
```json
{ "score": 0.8742, "explanation": "Excellent match (Very high semantic alignment). Shared keywords: java, sql." }
```

#### `POST /ocr`
```bash
curl -X POST http://localhost:5001/ocr \
  -F "file=@resume.pdf"
```
```json
{ "text": "Alice Johnson Senior Java Developer...", "ocr_used": false, "chars": 1850 }
```

### Java Backend

#### `POST /search`
```bash
curl -X POST http://localhost:8080/candidate-search/search \
  -H "Content-Type: application/json" \
  -d '{"query":"Python machine learning engineer with 3 years experience"}'
```
```json
{
  "query": "Python machine learning engineer with 3 years experience",
  "intent": { "skills": ["Python", "Machine Learning"], "experience_years": 3, "location": null },
  "results": [
    {
      "candidate": { "name": "Grace Lee", "currentRole": "ML Engineer", "location": "Los Angeles, CA", ... },
      "match_pct": 91,
      "explanation": "🟢 Excellent match. Semantic relevance: 94%. Matching skills: python, machine learning. 4.5 years exp meets 3 year requirement. Location: Los Angeles, CA."
    }
  ],
  "total_candidates_evaluated": 35
}
```

#### `POST /upload` (multipart/form-data)
```bash
curl -X POST http://localhost:8080/candidate-search/upload \
  -F "file=@resume.pdf" \
  -F "name=John Doe" \
  -F "skills=Java, SQL, Spring Boot" \
  -F "experience=4"
```

## 🔄 System Flow

```
Browser / API Client
     │
     ▼
SearchServlet (Java / Tomcat)
     │
     ├─► ClaudeService ──► Anthropic Claude API
     │         │                (intent extraction)
     │         ▼
     │     { skills, experience_years, location }
     │
     ├─► CandidateDAO ──► MySQL (candidates table)
     │
     └─► MatchScorer
              │
              ├─► Python /score ──► Sentence Transformers
              │      (60% weight)   (semantic similarity)
              │
              ├─► Skill overlap calculation (25% weight)
              └─► Experience proximity scoring (15% weight)
                       │
                       ▼
               Ranked Results + Explanation
```

## 🔧 Troubleshooting

| Problem | Fix |
|---------|-----|
| Claude API errors | Set `ANTHROPIC_API_KEY` in `.env`; fallback keyword parser activates automatically |
| PaddleOCR slow startup | Normal on first run (~2 min to download models) |
| MySQL connection refused | Ensure MySQL is running: `sudo service mysql start` |
| Tomcat 404 on /search | Check `candidate-search.war` is in `$TOMCAT_HOME/webapps/` and Tomcat has reloaded |
| `mvn` not found | Add Maven to PATH: `export PATH=$PATH:/opt/maven/bin` |
| OCR gives empty text | Ensure the image/PDF has sufficient resolution (≥300 DPI recommended) |
